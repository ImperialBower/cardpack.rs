# Update Log

## 2026-07-23
* **Examples flag-free — via a cargo alias, after reverting a self dev-dependency.** First attempt added `cardpack = { path = ".", features = ["full", "funky"] }` as a dev-dep; it broke `cargo deny check bans` in the `kernel-purity` job, because the dev-dep feature-activates `full`+`funky` on cardpack's own node in `cargo metadata`'s feature-unified resolve, so `colored`/`serde_norway`/`fluent-templates` appear as **normal** deps of the kernel. Neither `--exclude-dev`, `[graph] exclude-dev`, nor `no-default-features` suppresses it (`cargo tree --edges normal` stays clean the whole time — that divergence is the trap). Replaced with an `[alias] ex = "run --features full,funky --example"` in `.cargo/config.toml`, so `cargo ex <x>` needs no flags and the dependency graph is untouched. Restored the host `test --no-default-features --lib` and `build --no-default-features --examples` CI steps, which are meaningful again. New [decision](decisions/examples-flag-free-alias.md) (supersedes the deleted `examples-self-dev-dependency.md`); [feature flags](architecture/feature-flags.md) gotcha rewritten.
* **Feature graph**: `funky` now implies `serde` (`funky = ["std", "serde"]`) — the `funky/types/*` files use serde unconditionally, so `--features funky` alone left `serde` unlinked and the `buffoon`/`funky_tour` examples failed to compile ([feature flags](architecture/feature-flags.md), [funky engine](architecture/funky-engine.md)).
* **Decks**: Added Mughal (96) and Dashavatara (120) Ganjifa decks — [deck catalog](decks/deck-catalog.md) now covers 14 kinds; [localization](architecture/localization.md) gains the `mughal`/`dashavatara` fluent bases (EPIC-02).
* **Decision**: DeckKind is now #[non_exhaustive] and the crate bumped to 0.9.0 — adding enum variants was semver-major vs 0.8.1; future deck additions become non-breaking ([deck catalog](decks/deck-catalog.md)).
